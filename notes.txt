A collection of folders and files (with the folders having init.py) are present in rentals

1) models.py defines our models for databases -- see djando db models
2) services.py defines the logic that interacts with models.py and is called by our views.py (raise exceptions here)
3) views.py defines rest endpoints, what service methods they call, and their response (return status codes and exceptions if present)
4) I put my tests in a test folder but we could have just put it in tests.py
5) admin.py registers model views
6) urls.py defines urls and their connections to their views
7) serializers.py defines return serializers for easy json frontend reading

8) within the root urls.py make sure to add the "api/" endpoint and say which urls file it includes to (what app? in this case rentals)
9) /admin takes you to admin view where you can see db models and entries
